# تفعيل نظام نقاط الفولت العالي (خطوة بخطوة)

نظام النقاط جاهز بالكامل في الكود (الأيقونة ⭐، النوافذ، لوحة الإدارة).
البارح فقط تحتاج تنشئ قاعدة بيانات مجانية على Firebase (خدمة من جوجل)
وتربطها بالموقع بخطوتين بسيطتين. خذ وقتك، الخطوات مرتبة بالتفصيل.

---

## 1) إنشاء المشروع

1. افتح: https://console.firebase.google.com
2. سجّل دخول بحساب جوجل (Gmail) الخاص بك.
3. اضغط **Add project / إضافة مشروع**.
4. اكتب اسم مثل: `alfolt-alaly-points` → التالي → التالي → إنشاء المشروع.

## 2) تفعيل قاعدة البيانات (Firestore)

1. من القائمة الجانبية اليسرى: **Build → Firestore Database**.
2. اضغط **Create database**.
3. اختر **Start in production mode** → التالي.
4. اختر أقرب موقع سيرفر (مثلاً `eur3` أو `me-central1` إن كان متاح) → تفعيل.
5. بعد الإنشاء، اذهب لتبويب **Rules** فوق، واحذف كل المحتوى، والصق بدلاً عنه
   محتوى ملف `firestore.rules.txt` المرفق معك بالكامل، ثم اضغط **Publish**.

## 3) تفعيل تسجيل دخول الموظفين (Authentication)

1. من القائمة الجانبية: **Build → Authentication**.
2. اضغط **Get started**.
3. من قائمة طرق الدخول، فعّل **Email/Password** فقط (اضغط عليها → شغّل المفتاح → حفظ).
4. اذهب لتبويب **Users** بالأعلى → **Add user**.
5. أضف بريد وكلمة مرور لكل موظف (2-3 موظفين حسب طلبك). مثال:
   - `staff1@alfolt.com` / كلمة مرور قوية
   - `staff2@alfolt.com` / كلمة مرور قوية
   - يمكنك استخدام أي بريد إلكتروني حتى لو وهمي، المهم تتذكره أنت والموظف.

## 4) الحصول على بيانات الربط (Config)

1. ارجع للصفحة الرئيسية للمشروع (اضغط ⚙️ بجانب "Project Overview" أعلى اليسار → Project settings).
2. انزل لقسم **Your apps** واضغط أيقونة **</>** (Web).
3. اكتب اسم أي شيء مثل "الموقع" → Register app (لا تحتاج Firebase Hosting، تجاهله).
4. راح يظهر لك كود فيه كائن اسمه `firebaseConfig` يشبه هذا:

```js
const firebaseConfig = {
  apiKey: "AIzaSyxxxxxxxxxxxxxxxxxxxxxxx",
  authDomain: "alfolt-alaly-points.firebaseapp.com",
  projectId: "alfolt-alaly-points",
  storageBucket: "alfolt-alaly-points.appspot.com",
  messagingSenderId: "123456789012",
  appId: "1:123456789012:web:abcdef123456"
};
```

5. انسخ هذا الكائن كامل وابعثه لي، وأنا أحدّث ملف `loyalty.js` وأرسل لك
   النسخة النهائية جاهزة للرفع. (هذه البيانات ليست سرّية، آمن مشاركتها).

---

## بعد الربط: كيف يشتغل النظام؟

- **⭐ (بجانب الجرس):** يفتح قائمة فيها 5 خيارات.
- **🔍 تحقق من رصيدي:** مفتوح لأي زائر — يدخل رقمه ويشوف رصيده الحالي فقط
  (قراءة فقط، لا يقدر يسجّل نفسه ولا يعدّل شيء).
- **🆕 تسجيل عميل جديد:** للموظف فقط (يطلب تسجيل دخول). يُستخدم بعد التأكد
  من رقم العميل شخصياً عند الزيارة. لا يمكن تسجيل نفس الرقم مرتين، ويبدأ
  العميل دائماً برصيد صفر بالضبط.
- **⭐ تحصيل النقاط:** للموظف فقط — يدخل رقم العميل ومبلغ الفاتورة، والنظام
  يضيف نقطة عن كل 50 ريال تلقائياً. لو الرقم غير مسجّل، يرفض العملية ويطلب
  تسجيل العميل أولاً.
- **🎁 استبدال النقاط:** للموظف فقط — يدخل رقم العميل وعدد النقاط، ويرفض
  العملية تلقائياً لو الرصيد غير كافٍ.
- **⚙️ إدارة:** للموظف فقط — قائمة بكل العملاء وأرصدتهم، مع إمكانية تعديل أي
  رصيد يدوياً والبحث بالاسم أو الرقم.
- تسجيل دخول الموظف يبقى محفوظاً بالمتصفح حتى يضغط "تسجيل خروج".

## طبقات الحماية من الأخطاء والتلاعب

1. **تحقق صارم من صيغة الجوال** في كل نموذج: يجب أن يكون رقم سعودي بصيغة
   `05XXXXXXXX` بالضبط (10 أرقام)، وإلا يرفض النظام الإدخال برسالة واضحة.
2. **منع التسجيل المكرر:** لو الرقم مسجّل مسبقاً، تظهر رسالة تحذير باسم
   ورصيد العميل الحالي، ولا يتم إنشاء بطاقة جديدة فوقه.
3. **منع الرصيد السالب نهائياً:** حتى لو صار خطأ برمجي أو محاولة تلاعب،
   قاعدة البيانات نفسها (Firestore Rules) ترفض أي رصيد أقل من صفر.
4. **تعطيل زر الإرسال أثناء التنفيذ:** يمنع تكرار العملية بالخطأ عند
   الضغط المزدوج أو الإنترنت البطيء.
5. **سجل تدقيق ثابت:** كل عملية تحصيل/استبدال/تعديل تُسجَّل في `transactions`
   باسم الموظف الذي نفّذها فعلياً (موقّعة من طرف Firebase Auth نفسه، لا يمكن
   لأي شخص انتحال اسم موظف آخر)، ولا يمكن تعديل أو حذف هذا السجل أبداً.
6. **التسجيل حصراً من الموظف:** لا يوجد أي مسار يقدر فيه عميل يسجّل نفسه
   من بعيد، فيستحيل عملياً دخول أرقام وهمية للنظام.

## ملاحظات أمان مهمة

- لا تشارك كلمات مرور الموظفين مع أحد خارج الفريق.
- بيانات `firebaseConfig` ليست سرّية ويمكن أن تظهر في كود الموقع للعامة، الحماية
  الحقيقية موجودة في ملف الصلاحيات (`firestore.rules.txt`) الذي يمنع أي شخص
  غير مسجّل دخول من تعديل النقاط.
- الخطة المجانية (Spark) من Firebase تكفي لهذا الاستخدام بدون أي تكلفة، إلا إذا
  كبر عدد العملاء جداً (عشرات الآلاف يومياً).
